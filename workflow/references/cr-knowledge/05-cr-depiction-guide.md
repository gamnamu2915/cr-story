# CR 도구 묘사 가이드라인 (소설 창작용)

## 이 문서의 목적

소설에서 **뉴스 분석 도구**나 **분석 결과**가 등장할 때, 실제 CR-Check의 작동 방식을 반영하여 현실감 있게 묘사하기 위한 참고 자료입니다.

> **이 문서는 선택사항입니다.**
> CR 도구가 소설에 직접 등장하지 않는다면 참조할 필요가 없습니다.
> 소설가의 자유로운 상상을 제한하려는 의도가 아닙니다.

---

## CR-Check의 실제 작동 방식

### 핵심 특성

| 특성 | 실제 CR-Check |
|------|--------------|
| **분석 형식** | 구체적 문장 지목 → 해당 규범 인용 → 왜 문제인지 설명 |
| **리포트 구조** | 3개 대상별 (시민용 / 기자용 / 학생용) |
| **점수화** | ❌ 없음 |
| **통계적 처리** | ❌ 없음 (퍼센트, 카운팅 없음) |
| **개선안 제시** | ✅ "개선 전/후" 예시 형태 |
| **톤** | 비판적이지만 건설적, 교육적 |
| **성격** | "보조 도구", "확정적 판단 근거 아님" |

### 리포트 구조 예시

실제 CR-Check 리포트는 이런 구조를 갖습니다:

```
================================================================
                     [ CR-Check 분석 리포트 ]
================================================================

[ 분석 개요 ]
• 기사 제목: ...
• 매체명: ...
• 기사 유형: ...

----------------------------------------------------------------
  시민을 위한 종합 리포트
----------------------------------------------------------------

[구체적 문장 지목]
"정치권에 따르면 손인혁 헌재 사무처장은..."

[해당 규범 인용]
신문윤리실천요강 제3조 4항은 "출처가 분명하지 않거나
확인되지 않은 사실을 부득이 보도할 때는 그 사유를
분명히 밝혀야 한다"고 규정합니다.

[왜 문제인지 설명]
'정치권'이라는 모호한 출처 표시는 독자의 판단을
어렵게 만듭니다.

[개선안 제시]
• 개선 전: "정치권에 따르면 손인혁 헌재 사무처장은..."
• 개선 후: "5일 법사위 법안소위에서 손인혁 헌재
  사무처장은 공개 회의에서..."

----------------------------------------------------------------
  기자를 위한 전문 리포트
----------------------------------------------------------------

[건설적 비판의 톤]
"기자님의 기사는 복잡한 정치적 쟁점을 시의적절하게
다루고 있습니다. 그러나 몇 가지 개선이 필요한 부분이
있습니다..."

----------------------------------------------------------------
  학생을 위한 교육 리포트
----------------------------------------------------------------

[쉬운 설명과 질문 유도]
"여러분, 뉴스를 읽을 때 '이 정보는 누가 말한 거지?'라고
생각해본 적 있나요?"

================================================================
  ※ 활용 가이드
================================================================

이 리포트는 뉴스를 비판적으로 읽을 수 있게 돕는
'보조 도구'입니다. AI는 기사의 보도 관행 패턴을 감지해
연계되는 윤리규범과 함께 보여줄 뿐, 이를 확정적 판단의
근거로 삼을 수는 없습니다.
```

## 절대 하면 안 되는 묘사 (안티패턴)

### ❌ 통계적 처리

**잘못된 예시**:
```
화면에 분석 결과가 떴다.
"객관성 87% / 편향성 13%"
"문제 표현 3회 발견"
```

**왜 잘못인가**:
- CR-Check는 기사를 퍼센트나 횟수로 환원하지 않습니다
- 언론 윤리는 숫자로 계량화할 수 없는 영역입니다
- 이런 묘사는 도구의 성격을 왜곡합니다

### ❌ 점수화

**잘못된 예시**:
```
"이 기사의 신뢰도 점수: 47점 (등급 C)"
"윤리 준수율: 62%"
```

**왜 잘못인가**:
- 점수는 복잡한 저널리즘 현상을 단일 숫자로 단순화합니다
- "좋은/나쁜 기사" 이분법의 위험이 있습니다
- CR의 철학은 "판정"이 아니라 "질문 제기"입니다

### ❌ "검색창 + 클릭 = 즉각 판정" 단순화

**잘못된 예시**:
```
URL을 붙여넣고 '분석' 버튼을 누르자,
빨간 불이 켜졌다. "문제 있는 기사입니다."
```

**왜 잘못인가**:
- 도구가 심판처럼 "문제 있다/없다"를 판정하는 것이 아닙니다
- 결과는 "질문"이지 "판결"이 아닙니다
- 독자가 스스로 생각하게 하는 것이 목적입니다

## 올바른 묘사 방식

### ✅ 분석 결과 화면 묘사

**예시 1: 리포트를 읽는 장면**

```
화면을 스크롤했다. '시민을 위한 종합 리포트'라는 제목 아래,
기사의 특정 문장이 인용되어 있었다.

  "정치권에 따르면..."

그 아래 짧은 설명이 붙어 있었다.

  신문윤리실천요강 제3조 4항은 "출처가 분명하지 않거나
  확인되지 않은 사실을 부득이 보도할 때는 그 사유를
  분명히 밝혀야 한다"고 규정합니다. '정치권'이라는
  모호한 출처 표시는...

그는 자신이 쓴 문장을 다시 읽었다. 처음에는 아무 문제도
느끼지 못했던 그 문장이, 지금은 달라 보였다.
```

**예시 2: 개선안을 보는 장면**

```
리포트 하단에 '개선 제안'이 있었다.

  • 개선 전: "외부 로펌에 자문을 의뢰한 것으로 알려졌다"
  • 개선 후: "민주당 관계자는 '외부 로펌에 자문을 의뢰했다'고
    확인했다"

그녀는 두 문장을 번갈아 읽었다. 정보는 같았다.
하지만 책임의 소재가 달랐다.
```

### ✅ 캐릭터 반응 묘사

**예시 1: 기자의 자기성찰**

```
"나도 그렇게 써왔어."

수혁은 화면을 응시했다. 리포트는 그가 쓴 기사 세 개를
분석하고 있었다. 점수 같은 건 없었다. 대신 그가 습관처럼
썼던 표현들이 하나씩 지목되어 있었다.

'~로 알려졌다'
'~에 따르면'
'~라는 주장이 제기됐다'

그 옆에는 그가 스스로 외웠던 윤리강령 조항들이 인용되어
있었다. 모르는 내용이 아니었다. 그냥 까먹고 있었을 뿐.
```

**예시 2: 독자/시민의 반응**

```
"이거... 누가 만든 거야?"

민지는 리포트 맨 아래의 문구를 읽었다.

  ※ 이 리포트는 뉴스를 비판적으로 읽을 수 있게 돕는
  '보조 도구'입니다. 확정적 판단의 근거로 삼을 수는 없습니다.

"판사가 아니라 질문자인 거네."

그녀는 다시 기사를 읽기 시작했다. 이번에는 다르게 보였다.
```

**예시 3: 학생용 리포트를 보는 장면**

```
화면 아래쪽에 '학생을 위한 교육 리포트'라는 탭이 있었다.
클릭하자 말투가 달라졌다.

  여러분, 뉴스를 볼 때 "이 기사는 한쪽 이야기만 전하고
  있는 건 아닐까?"라고 의심해본 적 있나요?

준호는 피식 웃었다. 초등학생한테 설명하듯 쓰여 있었지만,
정작 자신도 그런 질문을 해본 적이 없었다.
```

### ✅ 규범 인용 장면

**예시: 등장인물이 규범을 확인하는 장면**

```
리포트가 인용한 조항을 검색했다.

  신문윤리실천요강 제3조 2항:
  "경합 중인 사안을 보도할 때 한 쪽의 주장을
  편파적으로 보도하지 않는다"

"우리가 정한 규칙이잖아."

그는 중얼거렸다. 외부에서 강요한 것이 아니었다.
한국기자협회, 한국신문협회. 기자들의 단체가 스스로 만든 것.
그런데 왜 안 지켜지는 걸까.
```

## 핵심 원칙 요약

### 1. CR은 판사가 아니라 질문자

> "이게 틀렸어!"가 아니라 "이건 왜 이렇지?"

도구는 판결을 내리지 않습니다. 질문을 던질 뿐입니다.
독자가 스스로 생각하게 만드는 것이 목적입니다.

### 2. 규범은 외부의 것이 아님

> "그들이 스스로 정한 규칙조차 안 지키잖아"

언론윤리규범은 시민이 강요한 것이 아닙니다.
기자협회, 신문협회 등 언론인들이 스스로 만든 약속입니다.
CR은 이 약속을 "환기"시키는 역할을 합니다.

### 3. 결과가 아니라 과정

> "리포트를 읽고 나서 기사가 다르게 보였다"

중요한 것은 "이 기사는 나쁜 기사다"라는 결론이 아닙니다.
독자가 스스로 질문하고 판단하는 과정이 핵심입니다.

### 4. 보조 도구의 한계

> "확정적 판단의 근거로 삼을 수는 없습니다"

CR-Check는 AI 기반 분석 도구입니다.
절대적 진실을 말해주는 것이 아니라,
생각할 거리를 제공하는 보조 도구입니다.

## 소설적 활용 예시

### 장면별 활용

| 장면 | 올바른 묘사 | 피해야 할 묘사 |
|------|------------|--------------|
| 분석 결과 확인 | "특정 문장이 인용되고, 관련 규범이 설명되어 있었다" | "신뢰도 47점, 등급 C가 떴다" |
| 기자의 반응 | "자신이 습관처럼 썼던 표현들이 지목되어 있었다" | "빨간 불이 켜지며 '문제 기사'라는 판정이 내려졌다" |
| 독자의 반응 | "다시 기사를 읽었다. 이번에는 다르게 보였다" | "이 기사가 87% 편향되었다는 걸 알게 되었다" |
| 갈등 장면 | "그들이 스스로 정한 규칙조차 안 지키잖아" | "AI가 너희 기사를 낙제점 줬어" |

### 대사 예시

| 상황 | 올바른 대사 | 피해야 할 대사 |
|------|-----------|--------------|
| 도구 소개 | "판결이 아니야. 질문을 던지는 거지." | "이걸로 좋은 기사인지 나쁜 기사인지 판별할 수 있어." |
| 결과 설명 | "네가 쓴 문장이 어떤 규범과 맞닿아 있는지 보여주는 거야." | "네 기사는 윤리 점수가 낮아." |
| 반박 | "보조 도구일 뿐이야. 확정 판단은 아니지." | "AI가 분석한 거니까 객관적인 거 아냐?" |
