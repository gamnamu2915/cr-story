# 멀티에이전트 소설 창작 시스템 - 실행 가이드

## 프로젝트 컨텍스트

이 시스템은 **CR(Checking the News Reliability) 프로젝트**의 일부입니다.

### 핵심 목표
언론 문제와 미디어 리터러시에 대한 **공감과 이해**를 이끌어내는 소설 창작.
직접적인 CR 홍보가 아니라, 독자가 스스로 질문을 던지게 하는 것이 목표입니다.

### 창작 원칙
- **프로파간다가 아닌 문학**: 해답을 제시하지 않고, 질문을 던진다
- **구체적 현실의 재현**: 추상적 비판보다 생생한 현장 묘사
- **열린 결말 지향**: 독자가 스스로 생각하게 만드는 여백

---

## 파이프라인 구조

```
[사용자 입력: 소재/주제]
    ↓
[Step 0] 리서처 → 도메인 지식 (선택)
    ↓
[Step 1] 철학자 → 화두/문제의식 도출
    ↓
[Step 2] 소설가 A → 시놉시스 작성
    ↓
[Step 3] 소설가 B → 시놉시스 비평
    ↓
[Step 4] 소설가 A → 피드백 반영 + 본문 초고
    ↓
[Step 5] 편집자 → 편집 의견
    ↓
[Step 6] 소설가 A → 편집 반영
    ↓
[Step 7] 비평가 → 문학적 해석
    ↓
[Step 8] 소설가 A → 비평 반영 + 최종본
    ↓
[Step 9] 터치업 → 문체 보강본 (선택)
```

## 에이전트 역할 요약

| 에이전트 | 역할 | 핵심 역량 |
|---------|------|----------|
| **리서처** | 도메인 지식 제공 | 내부자 시선, 암묵지, 다중 관점 |
| **철학자** | 화두/문제의식 도출 | 본질적 질문 추출, 철학적 깊이 부여 |
| **소설가 A** | 시놉시스/본문 작성 | 구조 설계, 캐릭터 창작, 감각적 표현 |
| **소설가 B** | 구조적 비평 | 플롯 분석, 개연성 점검, 건설적 피드백 |
| **편집자** | 문장 다듬기 | 간결성, 리듬, 가독성, 몰입도 강화 |
| **비평가** | 문학적 해석 | 주제의식 분석, 상징 해독, 승화 방향 제시 |
| **터치업** | 서술 단락 삽입 | 대화 흐름에 호흡 부여, 병목 생성 |

| **구조** | 짜임새, 정교함, 탄탄한 리듬, 완성도 | 소설가 A, 소설가 B |
| **몰입** | 흡입력, 다음 장이 궁금함, 단숨에 읽히는 에너지 | 소설가 A, 편집자 |
| **깊이** | 사안의 이면을 꿰뚫음, 철학적 사유, 묵직한 질문 | 철학자, 비평가 |
| **공감** | 살아있는 캐릭터, 깊이 있고 공감되는 인물 | 소설가 A, 소설가 B |

## 실행 방법

### 전체 프로세스 한 번에 실행
```
이 소설 창작 시스템의 전체 프로세스를 실행해줘.

1. inputs/input.md를 읽어줘
2. project_context.md를 읽어줘
3. prompts/ 폴더의 각 단계를 순서대로 실행해줘
4. 각 단계의 결과물을 outputs/[story-이름]/에 저장해줘
5. 각 단계 완료 후 간략히 보고해줘

도메인은 "언론/탐사보도"로 설정해줘.
```

### 단계별 실행
각 단계마다 해당 프롬프트 파일을 읽고 실행하도록 요청

## 재작성 모드

